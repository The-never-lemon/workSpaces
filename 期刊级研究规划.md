## 0. 顶层定位

### 0.1 论文标题候选

> *"Value-Aware Joint Coordination for Multi-Robot Exploration with Constrained Field-of-View"*

或更激进版本：

> *"From Implicit to Internalized Coordination: A Communication-Robust MARL Framework for Field-of-View-Constrained Exploration"*

### 0.2 一句话 pitch

> ICRA 2025 MARVEL 在多机器人探索中提出 attentive privileged critic 解决信用分配，但其**执行期协调依赖一套四件套的隐式机制**（独立采样 + occupancy ternary 信号 + 距离排序 reroute + 运动模型兜底），这套机制在 N_agents 偏离训练分布、通信受限、地图尺度变化时崩塌。本工作提出一个**价值驱动、可学习、可证明收敛的内生协调机制**，将协调能力直接嵌入策略；理论上证明与 SAC 价值函数对齐；实验上在 4 个 SOTA baseline、6 类分布外场景下一致优于 MARVEL。

### 0.3 期刊审稿人会问的三个问题

1. **Novelty**：MARVEL 已经有 attentive privileged critic 做信用分配，本工作的协调机制相比之下新在哪里？
2. **Rigor**：所提的"价值驱动协调"有理论保证吗？收敛性？最优性 gap？
3. **Generality**：只在 MARVEL setting 上有效吗？换成 ARiADNE、CAtNIPP 的 single-agent setting 还成立吗？

下文每个 contribution 都给出对这三个问题的答案。

### 0.4 不做真机的合理性辩护（论文 Limitations 部分要写）

- **算法贡献与机器人形态正交**：协调机制本身属于决策层，与 actuator/SLAM 等无关；
- **公平对比**：仿真允许在严格控制条件下与多个 SOTA 对比；
- **统计显著性**：5 种子 × 100 episode × 多配置（≥10⁴ runs），真机一般做不到；
- **ARiADNE [ICRA 2023]、CAtNIPP [ICRA 2023]** 在顶会发表时也是纯仿真，对齐传统；
- 论文 Section V.C *"Experimental Validation"* 已经做过真机演示，本工作沿用其结论即可，无需再做。

## 3. C1：Value-Aware Joint Coordination (VJC)

### 3.1 问题定义

设 $N$ 个 Agent，每个 Agent 在自己的局部图上有 $K=25$ 个候选邻居节点 × 3 个候选朝向，共 75 个候选动作。MARVEL 原版的协调流程是：

1. 每个 Agent **独立**从 $\pi_i(\cdot|s_i)$ 采样动作 $a_i = (v_j^i, h_q^i)$；
2. 若两个 Agent 选了同一个 $v_j$，**仅以欧氏距离排序**决定保留权；
3. 被剔除的 Agent **从 25 个最近邻中选第一个未被占用的节点**作为替代。

这个流程存在 5 个根本缺陷：

| # | 缺陷 | 影响 |
|---|------|------|
| ① | reroute 不看价值 | 替代节点的 $Q$ 或 utility 是 0 也会被选 |
| ② | reroute 不看朝向 | 朝向被强制保留，但目标节点变了——朝向变得"反方向" |
| ③ | 距离排序破坏 SAC 信用 | 离得远的 Agent 总是让步，长期看会有"二等公民"的退化策略 |
| ④ | 替代候选仅 25 个 | 实际可达节点可能有 360 个，最近邻 25 个不一定包含真正"该去"的节点 |
| ⑤ | 训练—执行一致但被动 | 训练时同样跑 reroute，但 actor loss 用调整后动作的 logp 更新，没有重要性采样校正 |

### 3.2 方法：VJC

**输入**：$N$ 个 Agent 当前观测 $\{s_i\}$、$N$ 个 Agent 在 $K \times 3 = 75$ 个候选动作上的 $Q_\theta(s_i, a) - \alpha \log \pi(a|s_i)$（共 $75N$ 个标量）。

**Step 1（cost 计算）**：对每个 Agent $i$、每个候选 $(v_j, h_q)$，定义：

$$c_i(v_j, h_q) = -\Big(\min(Q_{\theta_1}, Q_{\theta_2})(s_i, (v_j, h_q)) - \alpha \log \pi_i((v_j, h_q) | s_i)\Big)$$

**Step 2（联合分配）**：求解：

$$
\hat{\mathbf{a}} = \arg\min_{\substack{\mathbf{a} \\ \text{节点互斥, 朝向自由}}} \sum_i c_i(a_i)
$$

注意约束："节点互斥" = 不同 Agent 选不同 $v_j$；"朝向自由" = 同一节点不同朝向 OK，但因为节点互斥，朝向冲突也不会发生。

**Step 3（化归为标准 Hungarian）**：把 75 维候选空间投影到 $K=25$ 维节点空间：对每个 Agent $i$、每个节点 $v_j$，定义 $c_i^{\text{node}}(v_j) = \min_q c_i(v_j, h_q)$，得到 $N \times K_{\text{global}}$ 的代价矩阵（$K_{\text{global}}$ 是 N 个 Agent 候选节点的并集）。用 `scipy.optimize.linear_sum_assignment`（O((N+K)³)）求解。最后对每个 Agent 的分配节点回填最优朝向 $\hat{h}_i = \arg\min_q c_i(\hat{v}_i, h_q)$。

**Step 4（无偏策略梯度）**：保留两个动作流：

- `a_sampled = π_i 采样得到的动作`：用于 actor loss
- `a_executed = VJC 解 + 环境推进`：用于 critic loss 和环境

由于 actor loss 仅用 $\nabla \log \pi(a_{\text{sampled}}) \cdot (\alpha \log \pi - Q)$，且 $a_{\text{sampled}} \sim \pi$，策略梯度估计是无偏的（这是 SAC 标准形式）。critic 用 $(s, a_{\text{executed}}, r, s')$ 也合法（off-policy 性质）。

### 3.4 理论分析（命题 1+2，详见 C5）

- **命题 1 (最优性 gap)**：在 SAC 标准假设下，VJC 解的联合 Q 值与 oracle joint Q 最大值的差距上界为 $O(N \cdot \epsilon_{\text{cred}})$，其中 $\epsilon_{\text{cred}}$ 是 IQL 信用残差。
- **命题 2 (策略梯度无偏性)**：actor loss 的梯度估计无偏；critic loss 是合法的 off-policy 估计。

## 4. C2：Intent-Aware Self-Coordinating Policy (ISCP)

### 4.1 问题定义

C1 解决了协调的最优性问题，但仍是**中心化**——需要把 N 个 Agent 的 Q 值全部收集起来跑 Hungarian。这违反 MARVEL 论文的"decentralized execution"承诺，且在通信受限场景失效。

C2 的目标：**把 C1 学到的协调能力"压缩"进策略本身**，让每个 Agent 在本地决策即可逼近 VJC 的解，无需中心化求解器。

### 4.2 方法：双通道意图 + Self-Coordination Loss (SCL)

#### 4.2.1 双通道意图通信

每个 Agent 每决策步广播两条信息（每条 ≤ 256 字节）：

- **Channel A (位置意图)**：$(x_i, y_i, \theta_i)$ + 下一步目标 $(x_i', y_i', \theta_i')$
- **Channel B (价值意图)**：top-3 候选 $(v_j, h_q)$ 及其 Q 值

#### 4.2.2 Intent Attention 层

在 `PolicyNet.decode_state` 之后插入一个 cross-attention，query 是当前节点特征，key/value 是队友意图编码：

```
encoded_nodes = Encoder(node_inputs)              # 不变
intent_features = encode_intents(neighbor_intents) # 新增，128 维 / per channel
context = current_node_feature
context += IntentAttention(query=context, kv=intent_features)
pointer_logits = SingleHeadAttention(query=context, key=neighbor_features)
```

#### 4.2.3 Self-Coordination Loss (SCL)

辅助损失项，强制策略学到"避开队友意图"：

$$\mathcal{L}_{\text{SCL}} = \mathbb{E}\Big[\sum_{j \neq i} \pi_i(a_j^{\text{exec}} | s_i, \text{intent}_j) \cdot \mathbb{1}[v(a_j^{\text{exec}}) \in \mathcal{N}_i]\Big]$$

含义：当队友 $j$ 已经选定动作 $a_j^{\text{exec}}$，且该节点恰好也在 $i$ 的可选邻居中，那么 $i$ 选同一节点的概率应被惩罚。

总 actor loss：$\mathcal{L} = \mathcal{L}_{\text{SAC}} + \lambda_{\text{SCL}} \cdot \mathcal{L}_{\text{SCL}}$，$\lambda$ 从 0.1 衰减到 0.01。

#### 4.2.4 与 VJC 的关系

- **VJC 是 ISCP 的 oracle**：训练时 VJC 给出"理想分配"，ISCP 通过 SCL 拟合这个分配
- **部署时**：只用 ISCP，无 VJC——所以 ISCP 的实测性能上界是 VJC

### 4.3 理论分析（命题 3，详见 C5）

**命题 3 (无中心化协调的次优性)**：在 SCL 收敛到 0 且通信无延迟的极限下，ISCP 稳态分配等价于一个 Nash 均衡；该均衡解与 VJC 解的 expected utility 差距由 Q 估计误差 $\epsilon_Q$ 线性放大。

## 5. C3：Communication-Constrained CTDE (CC-CTDE)

### 5.1 问题定义

MARVEL 论文 Section III.A 显式假设 *"perfect communication between agents"*。这在仿真中是合理简化，但：

- **真实部署**：UAV 间通信距离有限（50–200 m）、室内 NLOS 导致 10–30% 丢包率
- **训练—部署 gap**：训练时全连通策略，部署时部分断连，策略行为不可预测
- **科研价值**：通信约束下的 multi-robot exploration 是一个相对未充分研究的方向

### 5.2 方法

#### 5.2.1 通信图域随机化

每个 episode 开始随机生成通信图 $\mathcal{G}_{\text{comm}}$：

- 基础边：物理距离 ≤ `COMM_RANGE`，`COMM_RANGE` 从 $[30, 100]$ m 均匀采样
- 随机 dropout：每条边以 $p_{\text{drop}} \sim U[0, 0.5]$ 失效
- 时变 dropout：每决策步重新采样

#### 5.2.2 缺失意图的 [MISSING] token

当 Agent $i$ 未收到 $j$ 的意图时，用一个可学习的 `[MISSING]` token 嵌入替代，与 ISCP 的 IntentAttention 兼容。

#### 5.2.3 局部 team reward

原版 team_reward 是全局共享：

$$r^{\text{team}}(t) = \frac{|\Delta \mathcal{F}_{\text{global}}|}{Z}$$

改为局部聚合（只统计自己 + 通信邻居的前沿减少）：

$$r_i^{\text{team}}(t) = \frac{|\Delta \mathcal{F}_i \cup \bigcup_{j \in \mathcal{N}_i^{\text{comm}}(t)} \Delta \mathcal{F}_j|}{Z}$$

这与 [Wang et al., AAMAS 2022] 的 *Local Cooperation Reward* 思路接近，但首次应用到 exploration。

### 5.3 代码改动清单

| 文件 | 改动 | 工作量 |
|------|------|--------|
| `utils/env.py` | `__init__` 增加 `comm_range, comm_dropout_prob`；新增 `get_comm_graph()` | 4 h |
| `utils/multi_agent_worker.py` | 每步基于 comm graph 决定可达 intent；reward 局部计算 | 5 h |
| `utils/model.py:IntentEncoder` | 增加 `[MISSING]` token embedding | 2 h |
| `parameter.py` | `COMM_RANGE_RANGE = (30, 100)`、`COMM_DROPOUT_RANGE = (0, 0.5)` | 0.5 h |

**总计 ~12 h（1.5 工作日）**

### 5.4 关键实验

横轴 `p_drop ∈ [0, 0.5]`，纵轴 explored_rate，三条线：

1. **MARVEL 原版**：通信断了但论文假设全连通 → 期望崩塌
2. **C1 (VJC, 全连通 oracle)**：上界
3. **C1+C2+C3**：本工作 → 期望接近 C1 上界

## 6. C4：Curriculum-based Cross-Scale Transfer (C³T)

### 6.1 问题定义

MARVEL Table I/II/III 只覆盖 N ∈ {2,4,8}、FoV ∈ {90°, 120°, 180°}、ds ∈ {10, 12, 15}。一个真正实用的协调机制应该能：

- **N 偏离训练分布**：训练 N=4，零样本部署到 N ∈ {2, ..., 16}
- **地图尺度变化**：训练 90×90 m，部署到 180×180 m 或 360×360 m
- **传感器配置混合**：同一团队不同 Agent 有不同 FoV / ds（异构）

### 6.2 方法

#### 6.2.1 课程式采样

```python
# 训练时每个 episode 采样
N_agents ~ Categorical([2: 0.1, 4: 0.35, 6: 0.25, 8: 0.15, 12: 0.10, 16: 0.05])
FoV ~ Uniform([60°, 180°])
ds ~ Uniform([8, 16]) m
map_scale ~ Uniform([1.0, 2.0])
```

#### 6.2.2 Set-encoded occupancy

替换原版 `occupancy ∈ {-1, 0, 1}` ternary scalar 为 set encoding：

```python
# 原版
occupancy[node_k] = -1 if self else 1 if any_other else 0

# 本工作（Set Attention）
occupancy[node_k] = SetAttention(
    set_of_robots_near_node_k,        # 包括位置、朝向、value-intent
    query=node_k_feature
)
```

这一改动让 occupancy 维度从 1 增长到 EMBEDDING_DIM=128，但**置换不变**（不依赖 Agent ID），可处理任意 N。

#### 6.2.3 Scale-invariant 坐标编码

把绝对坐标 $(x_i^k, y_i^k)$ 改为：

$$\text{enc}(d) = [\sin(d / 2^l \cdot \pi), \cos(d / 2^l \cdot \pi)]_{l=0,...,L-1}$$

类似 NLP 的 sinusoidal positional encoding，与绝对地图尺寸解耦。

#### 6.2.4 Adaptive NODE_PADDING_SIZE

原版固定 `NODE_PADDING_SIZE=360`。改为每 batch 内动态选择（取该 batch 最大节点数 + 32），与 Ray actor 兼容。

### 6.3 代码改动清单

| 文件 | 改动 | 工作量 |
|------|------|--------|
| `utils/env.py` | 支持 `map_scale`、Agent 异构 FoV/ds | 5 h |
| `utils/multi_agent_worker.py` | 每 episode 重采样 N、FoV、ds、map_scale | 3 h |
| `utils/node_manager.py` | occupancy set encoding | 6 h |
| `utils/model.py` | SetAttention 模块；sinusoidal 坐标 | 8 h |
| `parameter.py` | `N_AGENTS_DIST, FOV_RANGE, DS_RANGE, MAP_SCALE_RANGE, ADAPTIVE_PADDING` | 0.5 h |
| `driver.py` | 支持动态 padding | 3 h |

**总计 ~25.5 h（3-4 工作日）**

### 6.4 关键实验

零样本测试矩阵：$N \in \{2,4,6,8,12,16\} \times \text{map\_scale} \in \{1, 2, 4\}$ = 18 组配置，每组 100 episodes。

预期：原版 MARVEL 在 $N=12$ 或 map_scale=4 时 success rate < 70%；本工作维持 ≥ 90%。

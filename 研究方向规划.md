# 基于 MARVEL 的多机器人协同探索 —— 进一步优化框架与研究方向规划

> 本文档以 MARVEL（ICRA 2025）为 baseline，结合对源码（`utils/multi_agent_worker.py`、`utils/agent.py`、`utils/model.py`、`utils/node_manager.py`、`utils/utils.py`、`driver.py`）的实证审阅，提出一套可分阶段实施的优化框架，以及每一项研究方向所需的具体代码修改清单。

## 0. 总览：研究主线与 baseline 痛点映射

`MARVEL技术文档.md` 第 12、13 章已罗列 16+ 项工程/理论缺陷。这里把这些缺陷归并为 4 条研究主线，并按"科研价值 × 实施难度"排序：

| 主线 | 解决 baseline 哪些问题 | 科研贡献点 | 优先级 |
|------|----------------------|----------|--------|
| **R1. Q-aware 联合动作分配** | 12.1.2 / 12.1.3 / 13.4.1 / 13.4.4 | 把 Hungarian 升级为价值函数驱动的 K×3 联合分配 | ★★★★★ |
| **R2. 学习式协调器（替代外置 Hungarian）** | 13.4.3 / 12.3.5 / 12.1.4 | 把协调嵌入策略本身，end-to-end 训练 | ★★★★ |

下文每条主线给出：**(a) 问题重述 → (b) 改进方案 → (c) 具体代码修改 → (d) 实验设计**。

## 1. R1：Q-aware 联合动作分配（首选切入点）

### 1.1 问题重述

baseline 的 Hungarian 在 `utils/utils.py:joint_assignment` 中使用 `cost = -log π(a|s)` 作为代价。这存在三个根本问题：

1. **代价与价值脱钩**：SAC 的目标值是 `Q(s,a) − α log π(a|s)`，单独用 `−log π` 等价于 MAP 推断，与"团队累积回报最大"无关。
2. **节点与朝向解耦**：当前只对节点做指派，朝向独立采样。论文核心创新（heading-aware exploration）在协调层被丢弃。
3. **训练—执行不一致**：训练时 Hungarian 改写了动作，但策略梯度仍用调整后动作的 log-prob 更新 → off-policy 偏差。

### 1.2 改进方案

#### 1.2.1 升级代价函数：`cost = −(min(Q1, Q2)(s,a) − α log π(a|s))`
让 critic 参与分配决策，分配解 ↔ SAC 的目标函数对齐。

#### 1.2.2 K×3 联合分配
代价矩阵列数从 K 扩展到 K × NUM_HEADING_CANDIDATES = 75，约束改为：
- 每个 Agent 选一个 (node, heading) 对；
- 同一个 node 至多被一个 Agent 选中（heading 不约束，因为不同朝向覆盖不同区域）；
- 这是 bipartite matching with side constraints，仍可用 `linear_sum_assignment` 在扩展节点列上求解。

#### 1.2.3 显式校正策略梯度
保存两个动作：
- `a_sampled`：策略原始采样的动作（用于 actor loss）；
- `a_executed`：Hungarian 调整后实际执行的动作（用于环境推进）；

critic 用 `a_executed` 更新（off-policy 是合法的），actor 用 `a_sampled` 更新（保证 on-policy log-prob 一致性）。

| `utils/agent.py` | `get_node_action_scores` 改名为 `get_action_value_scores`，返回 `(q_values_per_action[K*3], logp_per_action[K*3])`；新增方法读取 critic 网络。注意：worker 端目前不持有 QNet，要么把 QNet 权重广播到 worker，要么用 target QNet。**推荐**：将 QNet 与 PolicyNet 一起作为 worker 输入。 |

## 2. R2：学习式协调器（消除外部 Hungarian）

### 2.1 问题重述

R1 仍把协调留给外部组合优化器。更彻底的方案是：**把协调机制嵌入策略本身**，让策略输出"考虑了队友意图的动作分布"。这样：

- 不需要在执行期跑 Hungarian（部署到真实硬件更简单）；
- 训练—执行严格一致，无策略梯度偏差；
- 协调能力可以被策略学到的 inductive bias 进一步泛化（例如未见过的 Agent 数）。

### 2.2 改进方案：Iterative Refinement Decoder（IRD）

灵感来自 multi-agent path finding 中的 PRIMAL2 与 NLP 中的 non-autoregressive 解码器：

```
Round 0:  每个 Agent 用普通 PolicyNet 输出 π_i^(0)(a|s)
Round 1:  把所有 Agent 的 π^(0) 的 top-1 动作作为"邻居意图"嵌入，重新过 decoder：
          π_i^(1)(a|s, neighbor_intents) = PolicyNet(s_i, intent_emb)
Round 2-T: 迭代直到 top-1 不再变化或达到 T=3
```

实现上是在现有 PolicyNet 的 decoder 之前插一个 **intent-attention 层**：query=自身当前节点特征，key/value=队友的 top-1 节点特征。整个过程可微，end-to-end 训练。

### 2.3 具体代码修改

| 文件 | 修改 |
|------|------|
| `utils/model.py` | 在 `PolicyNet.forward` 中引入 `intent_features` 可选参数；增加 `IntentAttention(nn.Module)`，结构与现有 Decoder 相同但 key/value 来自队友意图嵌入 |
| `utils/agent.py` | 新增 `get_observation_with_intent(intent_emb)`；`select_next_waypoint` 支持迭代 refinement |
| `utils/multi_agent_worker.py` | 删除 Phase 2 Hungarian，改为：循环 T 次，每次收集所有 Agent 的 top-1 节点 embedding，再次 forward 得到新分布；最后一次的分布即为采样源 |
| `parameter.py` | 新增 `NUM_INTENT_ROUNDS = 3`、`INTENT_EMBEDDING_DIM = 64` |
| `driver.py` | actor loss 计算时需 detach 队友意图（避免梯度跨 Agent 反传，否则 batched 训练时计算图爆炸）；用 `intent_emb.detach()` |

#### 关键设计要点

1. **意图嵌入是离散动作的 embedding 还是节点特征?**
   推荐：节点特征 (128 维)，因为节点本身已经包含 utility/前沿等丰富信息。
2. **梯度传播策略**：训练时只对最后一轮的 logp 计算梯度，前面 T-1 轮 detach。否则计算图 T 倍展开，显存吃不消。
3. **collator 的不变性**：意图嵌入应对队友排列不变（无 Agent ID 偏置），用 multi-head attention 自然满足。

### 2.4 实验设计

| 实验 | 设置 | 度量 |
|------|------|------|
| E2.1 IRD vs Hungarian | 同 R1 baseline | 探索率、重叠率、推理延迟 |
| E2.2 轮数消融 | T ∈ {1, 2, 3, 5} | 边际收益 |
| E2.3 泛化测试 | 训练 N=4，测试 N ∈ {2,4,6,8,12} | 期望 IRD 在 N=12 时 graceful degradation，Hungarian 在 N≤6 后开始恶化 |

## 3. R3：时序记忆与长视野规划

### 3.1 问题重述

baseline 中 PolicyNet 是无状态前馈，每步 forward 不依赖历史；γ=1 + 128 步上限造成 myopic 决策；`Node.visited` 只是 0/1 标志，无时序信息。这导致：

- Agent 可能反复访问最近刚去过的节点；
- 在大地图（地图规模 > NODE_PADDING_SIZE=360）下，远端历史信息被遗忘；
- Q-bootstrap 在回合上限被截断时引入偏差。

### 3.2 改进方案

#### 3.2.1 GRU/GTrXL 状态记忆

在 `PolicyNet.decode_state` 之后插入 GRU（每个 Agent 持有独立 hidden state，episode 内沿决策步演化）。优势：参数量小、训练稳定。

```
current_node_feature [B, 1, 128]
       │
       ▼
GRUCell(input=128, hidden=128)  ← h_{t-1}
       │
       ▼
contextual_current_feature → 后续指针网络
```

#### 3.2.2 节点级时序衰减

把 `Node.visited` 升级为 `Node.last_visit_step`，节点输入维度增加 `visit_recency = exp(-(t_current - t_last_visit) / τ)`，τ=20 步。NODE_INPUT_DIM 从 6 → 7。

#### 3.2.3 N-step return 与 PER

driver.py 中 SAC 的 TD target 改为 N=3 步 return：
```
y = ∑_{k=0}^{N-1} γ^k r_{t+k} + γ^N * (min Q'(s_{t+N}, a_{t+N}) − α log π(a_{t+N}|s_{t+N}))
```
配合 Prioritized Experience Replay（PER），buffer 按 TD-error 加权采样。

### 3.3 具体代码修改

| 文件 | 修改 |
|------|------|
| `utils/model.py` | `PolicyNet.__init__` 增加 `self.recurrent = nn.GRUCell(EMBEDDING_DIM, EMBEDDING_DIM)`；`forward` 增加 `hidden_state` 输入与输出 |
| `utils/agent.py` | 新增 `self.policy_hidden = None`；每个 episode reset，每步调用 `self.policy_hidden = ...`；episode_buffer 增加 hidden state 槽位（注意保存 detached 版本） |
| `utils/node_manager.py` | `Node` 类新增 `self.last_visit_step`；`update_graph` 接受 `current_step` 参数 |
| `utils/agent.py` | `get_observation` 中 `node_inputs` 增加第 7 维 `visit_recency` |
| `parameter.py` | `NODE_INPUT_DIM = 7`；新增 `VISIT_RECENCY_TAU = 20`、`N_STEP_RETURN = 3`、`USE_PER = True`、`PER_ALPHA = 0.6`、`PER_BETA = 0.4` |
| `driver.py` | 实现 N-step target 计算；用 `SumTree` 数据结构实现 PER（可引 `gymnasium`/`stable-baselines3` 的实现） |
| `utils/multi_agent_worker.py` | 在 episode 末尾把整段轨迹按 N-step 切片送入 buffer |

### 3.4 实验设计

| 实验 | 目的 |
|------|------|
| E3.1 GRU on/off | 验证时序记忆对"避免反复访问"的影响（统计单 episode 内同节点重访率） |
| E3.2 N-step ∈ {1,3,5,10} | 找到 bias-variance 最优点 |
| E3.3 PER on/off | 样本效率（达到 95% 探索率所需的 episode 数） |

## 4. R4：物理一致性与 sim-to-real 桥接

### 4.1 问题重述

baseline 的运动子步是 `np.linspace(start, end, sim_steps+1)[1:]` 端点直连，可能穿越未知/障碍单元；`compute_allowable_heading` 仅在最终态约束 yaw_rate，子步内无约束；这会让传感器在物理不可达位置采样，导致 sim2real gap。

### 4.2 改进方案

#### 4.2.1 子步沿 A* 路径插值

替换 `intermediate_cells = np.linspace(...)` 为基于 `path_coords` 的等弧长插值：

```python
def interpolate_along_path(path_coords, n_steps):
    cum_dist = np.cumsum([0] + [np.linalg.norm(p2-p1) for p1, p2 in zip(path_coords[:-1], path_coords[1:])])
    target_dists = np.linspace(0, cum_dist[-1], n_steps+1)[1:]
    return [interp_2d(path_coords, cum_dist, td) for td in target_dists]
```

#### 4.2.2 子步内的动力学约束

`compute_allowable_heading` 改为按子步循环：每个子步只允许 `yaw_rate * dt` 的转向，下一子步的初始朝向是上一子步的结束朝向。

#### 4.2.3 真实碰撞回退

子步中如果发现 cell 是 OCCUPIED 或 UNKNOWN，停止前进、记录 collision，给负奖励 -1。当前 baseline 完全没有碰撞检测。

#### 4.2.4 传感器噪声与域随机化

`sensor_work_heading` 中以 5% 概率把 free→unknown、以 1% 概率把 unknown→free，模拟测距噪声；FoV、SENSOR_RANGE 每 episode 在 ±10% 内随机扰动。

### 4.3 具体代码修改

| 文件 | 修改 |
|------|------|
| `utils/motion_model.py` | 新增 `compute_subsstep_trajectory(path_coords, start_heading, target_heading, n_substeps, velocity, yaw_rate)` |
| `utils/multi_agent_worker.py` | 替换 `np.linspace + smooth_heading_change` 的两处调用 |
| `utils/sensor.py` | 增加 `noise_prob` 参数 |
| `utils/env.py` | 新增 `domain_randomize()`，episode 开始时调用；`update_robot_belief` 增加碰撞返回值 |
| `parameter.py` | 新增 `COLLISION_PENALTY = -1.0`、`SENSOR_NOISE_PROB = 0.05`、`USE_DOMAIN_RANDOMIZATION = True` |

### 4.4 实验设计

## 5. 整合路线图

### 5.1 推荐实施顺序

```
Phase 1 (1-2 周): R1 实现与验证
  └─ 这是最小可行改进，能直接对标 baseline 的 test_comparison.md
  └─ 产出：1-2 个表格 + 简单消融，足够写一个 short paper 或 workshop

Phase 2 (2-3 周): R3 实现
  └─ GRU + N-step + PER 是 SAC 的标准升级，工程风险低
  └─ 产出：与 Phase 1 的组合实验，更完整的 ablation

Phase 3 (3-4 周): R2 实现（重点贡献)
  └─ 学习式协调器，原创性最强，替代 Hungarian
  └─ 产出：核心方法部分，与 R1 形成对比

Phase 4 (1-2 周): R4 实现
  └─ 工程性增强，为投稿到 ICRA/IROS 准备硬件验证素材

Phase 5 (1 周): 综合实验与消融
  └─ 把 R1+R2+R3+R4 的组合做成完整 ablation table
```

## 6. 风险与备选方案

| 风险 | 影响 | 备选 |
|------|------|------|
| R1：QNet 在 worker 上推理增加显存 | 18 个 worker × 2 QNet 显存爆炸 | 仅广播 target QNet，或者在 driver 端集中评估再下发 |
| R2：IRD 训练不稳定 | 多轮 forward 的梯度方差大 | 退化为 1 轮（pure decoder + intent attention），仍优于 baseline |
| R3：PER 在 4 Agent shared buffer 下采样有偏 | 同 episode 不同 Agent 经验耦合 | 用 multi-buffer，每个 Agent 独立 PER |
| R4：域随机化降低训练曲线 | 训练步数翻倍 | 用 curriculum：前 50% 训练无噪、后 50% 加噪 |

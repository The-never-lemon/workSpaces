# 缺陷诊断与切入点决策（v2 — Baseline 锁定为 `D:\工作区\MARVEL\`）

> 综合本仓库 `D:\工作区\workSpaces-master\` 中两份文档（`MARVEL技术文档.md` v2、`期刊级研究规划.md` v3）以及原版 baseline 源码（`D:\工作区\MARVEL\` 约 4800 行）、ICRA 2025 MARVEL 原论文（`D:\工作区\77-ICRA2025-MARVEL.pdf`）的完整审阅，本文回答两个问题：

## 1. 方法论

诊断遵循三原则：

1. **凭证可追溯**：每个缺陷都必须定位到 `D:\工作区\MARVEL\` 的具体代码行或论文段落，不基于推测；
2. **可量化排序**：用"致命度 × 攻击成本 × 创新空间"三维评估；
3. **与期刊规划闭环**：每个缺陷都给出对应的 contribution（C1–C5）。

**v2 与 v1 的关键差异**：v1 错误地把本仓库 Hungarian patch 当作 baseline，导致诊断框架失焦。本 v2 把 baseline 严格锁定为 ICRA 2025 论文同步发布的官方代码 `D:\工作区\MARVEL\`，所有诊断重新对齐。

## 2. 缺陷盘点（按致命度从高到低）

将 baseline 的 14 项细粒度缺陷归并为 **5 个层级**：

### 2.1 致命级（fatal）：影响协调机制的根本能力

| ID | 缺陷 | 凭证 | 后果 |
|----|------|------|------|
| **F1** | 协调机制完全依赖端到端涌现 | `D:\工作区\MARVEL\utils\multi_agent_worker.py:75-110` reroute；`agent.py` occupancy ternary；`model.py:368-380` agent_decoder | 无显式联合优化层；分布外（N=12、p_drop>0、map_scale=4）时无 fallback |
| **F2** | 朝向脱离协调 | reroute 仅替换节点；朝向强制保留 | 目标变更后朝向变为"反方向"，违反 MARVEL heading-aware 核心论点 |
| **F3** | 距离排序破坏 SAC 信用对称 | `multi_agent_worker.py:92` `np.argsort(dist_list)` | 远距离 Agent 长期让步 → 习得"主动放弃高价值节点"的退化策略 |

**这三个 F 级缺陷构成一个完整故事链**：原版协调依赖隐式机制 → 隐式机制在分布外失效 → 即使表面"工作"也存在 SAC 信用层面的退化倾向。**这是 C1 VJC 的攻击靶点**。

### 2.2 严重级（severe）：影响泛化能力与适用范围

| ID | 缺陷 | 凭证 | 后果 |
|----|------|------|------|
| **S1** | occupancy {-1, 0, 1} 信息容量不足 | `agent.py:202` 中 `node_occupancy` 为 scalar；论文 Section IV.A | 多 Agent 扎堆时丧失分辨能力 |
| **S2** | 全连通通信硬假设 | 论文 Section III.A *"perfect communication"*；`env.py:robot_belief` 共享数组 | 通信受限场景 zero-shot 失败 |

## 4. 切入点决策：从 F1+F2+F3 → C1 VJC

### 4.1 为什么必须从 C1 开始？六条理由

**1) 凭据最硬，不依赖任何"补丁"对比**
v1 规划误以 Hungarian patch 为基线，证据强度依赖 `results/test_comparison.md`（已知 baseline 错位）。v2 直接以原版 reroute 为对手，**所有对比变量明确**：cost 函数（{节点距离, log π, Q, Q-α log π}）、协调维度（{节点, 节点+朝向}）。

**2) 工作量最小（18h，~2 工作日）**
- 原版 QNet 已逐动作输出
- 仅需新增 `joint_assignment_vjc()` + 修改 worker Phase 2
- 复用现有 SAC 训练管线，无需改 driver 损失函数

**3) 理论上有 oracle 上界**
集中式 VJC 是联合分配的最优解（一阶段意义下）。同样用 Hungarian 求解器，对比 4 种 cost：
- 节点距离（论文 baseline）
- log π（v1 的 Hungarian patch）
- Q only
- Q − α log π（本工作）

差异完全来自 cost 定义。**控制变量极其干净**，审稿人无懈可击。

**4) C2/C3/C4 都需要 C1 作为基础**
- C2 ISCP 的"价值意图通道"需要 worker 端有 Q 估计
- C3 CC-CTDE 的"局部 team reward"前提是 critic 价值估计已对齐
- C4 C³T 的 Set Attention 用 value 作为节点级 query

**没有 C1，C2/C3/C4 都是空中楼阁。**

**5) Novelty 论证直接**
> "MARVEL's coordination via distance-sorted reroute does not leverage the critic. We propose to use $Q - \alpha \log \pi$ as the joint assignment cost, which is consistent with the SAC objective. Empirically, this yields ΔX pp on N=12 explored rate."

这种"一行代码替换 + 显著提升"是审稿人最爱的 "low-hanging fruit with high impact"。

**6) 失败风险最低**
- 论文 Table I 表明 N=2/4/8 都 100% 成功——C1 不需要"打败 100%"，只需在 N≥12 或 ablation 上有提升即可
- 即使 C1 提升不显著，可以作为 ablation 控制项，证明"价值对齐是 C2 ISCP 的必要前置"，铺路 C2

### 4.2 决策树：C1 之后怎么走？

```
                  ┌──── 显著提升 (>5pp) ────→ C1 成为核心贡献
                  │                            进入 C2 ISCP
                  │
   C1 VJC 实验 ───┤
   (Week 1-3)     │
                  │                          ┌── 通信受限场景突出 → 直接做 C3 (跳过 C2)
                  └──── 提升较小 (<5pp) ────┤
                                             └── 跨规模场景突出 → 直接做 C4 (跳过 C2)
                                             
                                             两者都不突出 → 重新审视 C2 ISCP 的 SCL 设计
```

### 4.3 C1 的最小可行验证（Minimum Viable Validation, MVV）

**1 天**完成，作为 go/no-go 决策点：

| 步骤 | 操作 | 工作量 |
|------|------|-------|
| 1 | 把 `D:\工作区\MARVEL\` 整体拷贝到 `D:\工作区\workSpaces-vjc\` 作为新基线 | 0.5 h |
| 2 | 在 `utils/utils.py` 新增 4 个 `joint_assignment_*` 函数：`logp`, `q_only`, `q_minus_alpha_logp` 三种 cost + reroute（保留原版） | 2 h |
| 3 | 在 `utils/agent.py` 新增 `get_q_per_action()`（直接调用原版 QNet） | 1 h |
| 4 | 在 `utils/multi_agent_worker.py` 加 `COORDINATION_MODE` 切换 | 1 h |
| 5 | 在 `maps_test` 上跑 N=6 / FoV=120° / 50 episodes × 4 mode | 2 h（CPU/GPU 看资源） |
| 6 | 看 success_rate / overlap_ratio / explored_rate 四组对比 | 0.5 h |

**Go 判据**：`Q − α log π` 的 success rate ≥ `reroute` 的 success rate（持平即可），且 overlap_ratio 不变差。
**No-go 判据**：`Q − α log π` 显著退化（success rate 下降 >5pp）→ 可能 QNet 估计在分布外噪声太大，先在 N=4 重新训一遍 QNet，再做 MVV。

## 7. 结论：三句话答案

1. **以原版 MARVEL 为 baseline，最致命的缺陷**：F1 / F2 / F3 三者构成的"协调机制无显式联合优化层 + 朝向脱离协调 + 距离排序破坏 SAC 对称"三角问题。其他所有缺陷（S/M/E 级）都是这个核心问题在不同维度上的衍生表现。

2. **下一步应该从哪里切入**：**C1 VJC**（Value-Aware Joint Coordination）。理由：凭据最硬（无 Hungarian patch 错位）、工作量最小（18h，原版 QNet 已逐动作）、有 oracle 上界、是 C2-C5 的前置、novelty 论证直接、失败风险最低。

3. **第一周该做什么**：实施 §4.3 的 MVV，1 天内完成 4 种 cost mode 对比，凭数据决定全力推进 C1 还是先训独立 QNet。
